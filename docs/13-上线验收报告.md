## 五、问题处理记录

### 1. 数据导入问题

问题现象：CloudBase 控制台导入 JSON 时提示字段名不合法。  
处理结果：去除 `_id` 字段，改为由 CloudBase 自动生成主键，并补充适合控制台导入的 JSON 数据版本。

### 2. 云函数依赖问题

问题现象：单独上传云函数目录时，公共依赖目录未随函数一并上传。  
处理结果：将 `common/` 复制到每个云函数目录中，使函数具备独立部署能力。

### 3. 匿名访问权限问题

问题现象：前端通过匿名登录调用云函数时返回 `PERMISSION_DENIED`。  
处理结果：临时将云函数调用规则设置为测试期开放策略，保证前后端联调顺利完成。

### 4. 静态站点部署路径问题

问题现象：CloudBase 后台仍使用旧的 `./dist` 自定义部署命令，导致路径不存在。  
处理结果：改为直接上传本地 `dist` 构建产物，并取消错误的自定义部署路径。

### 5. 项目详情与详情页白屏问题

问题现象：部分页面因数据字段缺失或热更新状态残留导致空白。  
处理结果：补充空值兜底逻辑，重启开发服务器，并完善加载态、错误态和空状态显示。

## 六、性能与可维护性优化结果

本轮收口中已完成以下优化：

- 路由页面按需加载，减少首屏一次性加载体积
- Vite 配置增加 `manualChunks`，将 React、Router、CloudBase SDK 与其他依赖分块输出
- 统一 `LoadingState`、`ErrorState`、`EmptyState`
- 服务层统一走 `apiClient`，为后续扩展与维护保留清晰边界
