# 测试方案

## 测试目标

保证项目能演示、能答辩、能上线。

## 测试范围

- 页面显示是否正常
- 表单提交是否正常
- 数据读取是否正常
- 云函数是否返回正确结果
- 错误提示是否清晰
- 加载状态和空状态是否清晰

## 测试环境

- 本地开发环境：macOS + Node.js + npm + Vite
- 云端运行环境：Tencent CloudBase
- 浏览器：Safari、Chrome

## 功能测试用例

### 1. 首页

- 访问首页
- 检查平台标题、入口按钮、主视觉是否正常显示

### 2. 思想熔炉

- 查看猜想列表是否成功加载
- 打开猜想详情页
- 发布新猜想
- 提交评论

### 3. 研究工作室

- 查看项目列表
- 打开项目详情页
- 提交项目讨论

### 4. 用户相关

- 打开通知中心
- 打开个人中心
- 打开学者网络

## 联调测试

- 前端切换到 `cloudbase` 模式后能否正常读取数据
- 云函数是否存在权限错误
- 云数据库示例数据是否成功返回
- 页面是否存在因空字段导致的白屏

## 部署测试

- `npm run build` 是否成功
- CloudBase 静态网站是否能访问
- 云函数是否全部部署成功
- 子路由刷新后是否仍能正常访问

## 验收标准

- 主要页面能正常打开
- 主要交互链路能跑通
- 线上与本地效果基本一致
- 无持续性红色报错
- 具备截图留档与答辩演示条件
