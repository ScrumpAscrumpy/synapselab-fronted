# 答辩 PPT 讲稿版

## 第 1 页 题目页

各位老师好，我的毕业设计题目是《基于 React + CloudBase 的跨学科科研协作平台 SynapseLab 的设计与实现》。本课题面向跨学科科研协作场景，重点解决科研猜想交流、项目推进、研究者连接与通知反馈等问题。

## 第 2 页 研究背景

当前很多科研问题具有明显的跨学科特征，但实际协作中仍普遍依赖微信、邮件和共享文档等分散工具，存在信息沉淀不足、研究过程不可视、协作路径不清晰等问题。因此，我希望设计一个更适合跨学科研究团队使用的 Web 协作平台。

## 第 3 页 研究目标

本项目的目标是构建一个集科研猜想发布、研究项目展示、学者网络连接、通知反馈和个人中心管理于一体的平台，并完成从本地开发到云端部署的完整实现过程。

## 第 4 页 需求分析

根据业务分析，平台的核心需求包括五部分：一是科研猜想交流，二是项目协作推进，三是研究者之间的连接，四是通知反馈，五是个人资料和参与情况展示。

## 第 5 页 系统架构

系统采用三层架构。前端使用 React 构建页面，后端使用 CloudBase 云函数处理业务逻辑，数据层使用 CloudBase 云数据库存储平台数据。这样的结构使前后端职责比较清晰，也便于后续扩展。

## 第 6 页 技术选型

前端选择 React 和 Vite，是因为组件化开发和开发效率较高；后端选择 CloudBase，是因为它集成了静态网站托管、云函数和数据库能力，适合毕业设计项目快速完成全栈实现与部署。

## 第 7 页 核心模块

平台主要包括首页、思想熔炉、研究工作室、学者网络、通知中心和个人中心六个核心模块。其中思想熔炉负责承载科研猜想，研究工作室负责承载正式项目，学者网络用于连接研究者。

## 第 8 页 数据库设计

数据库主要包括 `users`、`ideas`、`projects`、`comments`、`project_discussions` 和 `notifications` 六个核心集合。它们分别用于支撑用户信息、科研猜想、研究项目、评论讨论和通知动态等功能。

## 第 9 页 系统实现

在系统实现过程中，我先基于 mock 数据完成前端页面和交互，再通过统一服务层切换到 CloudBase 真实环境。这样可以分离页面开发和云端联调，降低开发复杂度。

## 第 10 页 关键实现亮点

本项目的几个关键实现点包括：统一服务层设计、云函数按业务模块拆分、页面加载态和错误态设计、真实 CloudBase 上线部署，以及为非技术背景使用者补充的完整 README 与部署文档。

## 第 11 页 测试结果

测试结果表明，首页、思想熔炉、研究工作室、学者网络、通知中心和个人中心均可正常访问；评论、讨论和发布猜想等主要功能可以正常使用；系统也已经成功部署到 CloudBase 云端环境。

## 第 12 页 部署结果

本项目不仅完成了本地运行，还完成了数据库初始化、云函数部署、静态网站托管和线上访问验证，说明系统已经具备完整的软件演示条件。

## 第 13 页 总结

通过本次毕业设计，我完成了一个基于 React + CloudBase 的跨学科科研协作平台，实现了从需求分析、系统设计、前后端开发到云端部署的完整流程。项目已经具备毕业设计所需的软件工作量和演示效果。

## 第 14 页 展望

未来可以继续完善正式登录、权限细化、AI 分析能力、通知联动和成员协作流程等内容，使平台进一步向真实科研协作产品方向演进。
