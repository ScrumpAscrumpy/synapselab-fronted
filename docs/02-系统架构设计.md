# 系统架构设计

## 总体结构

系统分为三层：

1. 表现层：React 前端页面
2. 业务层：CloudBase 云函数
3. 数据层：CloudBase 云数据库

## 数据流

1. 用户打开前端页面
2. 前端调用云函数或数据库接口
3. 云函数处理业务逻辑
4. 数据库存储或返回结果
5. 前端更新界面

## 设计原则

- 前后端职责清晰
- 页面先跑通，再接真实数据
- 文档和代码同步整理
- 所有模块为 CloudBase 部署预留接口

## 当前实现状态

目前系统已经形成了一个适合后续接 CloudBase 的过渡架构：

1. React 页面通过 `services/` 访问统一 API 层
2. API 层根据配置选择 `mock` 或 `cloudbase`
3. 当前默认使用 `mock`
4. `cloudfunctions/` 中已经建立 `ideas`、`projects`、`users` 三类函数骨架
5. 云函数通过 `common/repository.js` 访问数据库或回退到 mock 数据
6. 前端通过 `web/src/lib/cloudbase.js` 初始化 CloudBase SDK

这样做的好处是：

- 前端页面先稳定
- 后续切换到 CloudBase 时改动集中在服务层
- 云函数目录已经和业务模块一一对应
