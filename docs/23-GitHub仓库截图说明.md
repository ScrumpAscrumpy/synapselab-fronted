# GitHub 仓库截图说明

这份文档用于指导 SynapseLab 的 GitHub 仓库展示，方便后续补充真实项目截图时保持统一风格。

## 建议放置的截图

建议在仓库中准备 4 到 6 张截图，优先展示下面这些页面：

1. 首页
说明文案：
用于展示 SynapseLab 的平台定位、核心理念和整体视觉风格。

2. 思想熔炉
说明文案：
用于展示科研猜想的发布、浏览、详情查看和 AI 辅助分析能力。

3. 研究工作室
说明文案：
用于展示跨学科项目的阶段推进、成员协作和项目管理能力。

4. 学者网络
说明文案：
用于展示学者、项目和论文节点构成的动态关系图谱，是本项目的核心特色页面之一。

5. 全屏图谱模式
说明文案：
用于展示沉浸式关系网络浏览效果，更适合作为答辩和项目展示亮点。

6. CloudBase 部署结果
说明文案：
用于展示系统已完成真实云端部署，而不是仅停留在本地开发阶段。

## 建议的截图命名

建议后续把截图统一放在 `docs/assets/screenshots/` 下，并按下面方式命名：

- `home.png`
- `ideation.png`
- `studio.png`
- `network.png`
- `graph-mode.png`
- `cloudbase-online.png`

## README 中的展示建议

如果后续把真实截图放进仓库，README 里可以增加一个“界面预览”小节，优先展示：

- 首页
- 思想熔炉
- 学者网络

这样访客在进入仓库时，能最直接理解这个项目的产品方向和完成度。
