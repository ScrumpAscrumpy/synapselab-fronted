# 数据库设计

## 设计目标

SynapseLab 的数据库需要同时支持：

1. 用户资料展示
2. 思想熔炉的猜想发布与浏览
3. 研究工作室项目管理
4. AI 分析结果存档
5. 评论、通知、贡献度统计

## 第一版核心集合

- `users`
- `ideas`
- `projects`
- `project_discussions`
- `comments`

## 扩展集合

- `notifications`
- `contributions`
- `badges`
- `ai_logs`
- `invites`

## 为什么这样设计

### 用户信息快照

在 `ideas` 中保留 `authorName` 和 `authorRole`，是为了避免每次列表展示都必须再次连表读取用户集合。

### 时间字段统一

大部分集合都带 `createdAt` 和 `updatedAt`，这样后续做排序、筛选、统计会更方便。

### 先满足论文和演示

第一版字段设计优先保证：

- 页面能展示
- 云函数好写
- 论文章节好解释
